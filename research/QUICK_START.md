# BDF 输入格式研究快速开始

## 🚀 立即开始

### 步骤 1：收集 BDF 输入文件示例

**如果您已有 BDF 输入文件**：
1. 将文件复制到 `research/bdf_examples/` 对应目录
2. 按照计算类型分类（energy、optimize、frequency 等）

**如果您需要获取示例**：
1. 查看 BDF 用户手册中的示例章节
2. 从 BDF 测试用例中提取
3. 从已有的计算任务中收集
4. 询问 BDF 用户社区

### 步骤 2：分析第一个示例

选择一个简单的示例（如单点能计算）开始分析：

1. **打开文件**：查看 BDF 输入文件内容
2. **识别结构**：
   - 文件格式（文本、行格式等）
   - 注释方式
   - 关键字格式
3. **记录发现**：在 `research/notes/structure_analysis.md` 中记录

### 步骤 3：填写映射表

根据分析的示例，开始填写映射表：

1. **方法映射**：`research/mapping_tables/method_mapping.yaml`
2. **基组映射**：`research/mapping_tables/basis_mapping.yaml`
3. **关键词映射**：`research/mapping_tables/keyword_mapping.yaml`

### 步骤 4：记录发现

在 `research/notes/findings.md` 中记录：
- 重要发现
- 遇到的问题
- 待确认事项

## 📋 分析检查清单

分析每个 BDF 输入文件时，检查以下内容：

### 基本结构
- [ ] 文件格式（文本、编码）
- [ ] 注释方式
- [ ] 关键字格式（大小写、前缀等）

### 分子结构
- [ ] 坐标输入格式
- [ ] 单位指定方式
- [ ] 电荷和自旋指定

### 计算方法
- [ ] 方法指定方式
- [ ] 基组指定方式
- [ ] 参数设置方式

### 计算类型
- [ ] 任务类型指定
- [ ] 相关参数设置

## 💡 分析技巧

1. **从简单开始**：先分析最简单的单点能计算
2. **对比分析**：对比不同计算类型的文件，找出共同点和差异
3. **逐步深入**：先理解基本结构，再研究高级特性
4. **及时记录**：发现任何信息都及时记录，避免遗忘

## 🔍 需要重点关注的问题

1. **坐标格式**：这是最关键的，需要准确理解
2. **方法映射**：不同方法在 BDF 中的具体指定方式
3. **基组映射**：基组名称和库文件路径
4. **参数格式**：各种参数的格式和单位

## 📝 下一步

1. 收集至少 3-5 个不同类型的 BDF 输入文件示例
2. 分析这些文件的结构
3. 开始填写映射表
4. 记录所有发现和问题
